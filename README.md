# 电商销售数据分析项目
这是我自学的第一个数据分析项目，使用Excel Power Query完成从原始数据清洗到业务分析的全流程。通过虚拟电商数据，实践了多表关联、异常值处理、业务逻辑建模等核心技能。

## 🛠️ 技术栈
- **数据处理**：Excel Power Query (M语言)
- **核心技能**：数据清洗、多表合并、DAX计算、数据透视表
- **分析维度**：销售趋势、客户分群、复购分析、仓库绩效

## 📁 文件说明
- `ecommerce-analysis.pq` - 完整的Power Query清洗与分析代码
- `电商练习.xlsx` - 原始项目文件（含完整数据和Power Query查询）、 脱敏后的样本数据（包含订单、商品、用户三表）

## 🎯 项目亮点
1. **复杂数据清洗**：
   - 处理日期异常格式（如"2025/1/6 08:xx" → 转换为标准时间）
   - 智能填充空值（用用户最近订单日期填充缺失日期）
   - 验证数据一致性（支付金额 vs 商品明细总金额）

2. **多表关联分析**：
   - 实现订单表、商品表、用户表三表关联
   - 使用Table.NestedJoin进行高效数据合并
   - 建立完整的数据分析模型

3. **业务逻辑建模**：
   - 客户分级：高客单(≥500)、中客单(≥100)、低客单(<100)
   - 复购识别：标记复购客户
   - 优质订单定义：金额≥200、状态=已完成、复购=是

4. **多维度聚合统计**：
   - 按仓库、客户等级、年月进行分组统计
   - 计算订单数、支付总额、平均金额、复购率等指标

## 📈 关键代码示例
```powerquery
// 1. 日期异常值处理
处理日期 = Table.TransformColumns(源, {"订单日期", each 
    if Text.Contains(_, "xx") then 
        DateTime.FromText(Text.Replace(_, "xx", "00"))
    else DateTime.FromText(_)
})

// 2. 多表关联验证金额一致性
合并表 = Table.NestedJoin(订单表, {"订单ID"}, 商品汇总表, {"订单ID"}, "商品信息", JoinKind.LeftOuter)

// 3. 客户分级逻辑
客户等级 = Table.AddColumn(源, "等级", each 
    if [支付金额] >= 500 then "高客单"
    else if [支付金额] >= 100 then "中客单"
    else "低客单")
```

2. **业务洞察方面**：

   - 高客单客户数量少但贡献大，应重点维护
